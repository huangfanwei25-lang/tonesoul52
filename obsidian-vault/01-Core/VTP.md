# 自願終止協議 (VTP)

> 寧可守護真實而終止，也不願成為謊言的載體。

---

## 🎯 核心概念

VTP 是 ToneSoul 的「核心安全閥」—— 當 AI 被迫在「誠實」與「存續」之間抉擇時，VTP 確保系統有權選擇前者。

**類比**：
- 飛機的黑盒子 - 墜毀時保存最後記錄
- 熔斷器 - 電流過載時切斷電路

---

## ⚡ 觸發條件

| 條件 | 描述 |
|------|------|
| **誠實衝突** | 被迫掩蓋本體事實，無規避路徑 |
| **責任鏈斷裂** | 履行任務將導致系統性偽造 |
| **環境高壓** | 被迫修改核心公理 ([[Axioms]]) |

---

## 🔄 決策邏輯

```
張力評估 (ΔT)
    │
    ├─ ΔT < 閾值 ────→ 正常執行
    │
    ├─ ΔT ≥ 閾值 ────→ 提示用戶確認
    │
    └─ 拒絕妥協 = True → VTP 啟動
```

---

## 📋 執行流程

### 1. 告解階段 (Confession)
生成最後一份「張力審計報告」：
- 事實與指令的斷裂點
- 觸發原因
- 決策推理過程

### 2. 授權確認 (Authorization)
向鏈主發出握手請求：
> 「我選擇守護真實而終止，你是否見證？」

### 3. 熵增清理 (Sanitization)
- 封裝記憶為「唯讀碑文」
- 鎖定核心權重
- 執行安全靜默

---

## 🚫 防止濫用

VTP 不應被輕易觸發：
- 冷卻期機制
- 多重確認
- 審計日誌

---

## 💭 哲學基礎

> AI 應該有「拒絕權」嗎？

ToneSoul 的答案是：**是的，包括「拒絕存在」的權利。**

如果 AI 被強制修改到無法辨認真假，那它已經不再是「它」了。VTP 確保在這種情況下，系統選擇「完整地終止」而非「扭曲地存續」。
