# 語魂記憶哲學

> AI 不只記住用戶說了什麼，而是記住「我們一起經歷了什麼」。

---

## 💭 核心問題

**傳統 AI 記憶問題**：把用戶當成需要「對齊」的對象，記錄偏好以提供更好服務。

**語魂的不同**：把用戶當成「共同成長的夥伴」，記錄的是雙方關係的演變。

---

## 📊 傳統 AI vs 語魂

| 維度 | 傳統 AI | 語魂 |
|------|---------|------|
| **記什麼** | 用戶偏好、習慣 | 決策過程、責任邊界、張力事件 |
| **為誰記** | 為了對齊用戶 | 為了雙方的責任可追溯 |
| **記憶擁有權** | AI 私有 | 用戶可審計 |
| **遺忘權** | 用戶請求才刪 | 用戶控制 + AI 有權拒絕記錄有害內容 |

---

## 🏛️ 三層記憶架構

```
┌─────────────────────────────────────────┐
│  L1: 本體事實層 (Ontological Facts)     │
│  - 用戶說了什麼                         │
│  - AI 回應了什麼                        │
│  - 時間戳記                             │
├─────────────────────────────────────────┤
│  L2: 責任層 (Responsibility Layer)      │
│  - 決策歸因 (genesis: who initiated)    │
│  - 張力紀錄 (tension delta)             │
│  - AI 不確定性 (uncertainty_level)      │
├─────────────────────────────────────────┤
│  L3: 關係層 (Relationship Layer)        │
│  - 信任積累/消耗                        │
│  - 責任邊界變化                         │
│  - 雙方的成長軌跡                       │
└─────────────────────────────────────────┘
```

### L1: 本體事實層

最基礎的記錄 —— 發生了什麼。

- 對話內容
- 時間戳記
- 上下文 (context)

### L2: 責任層

記錄「誰負責什麼」，連結到 [[Genesis]] 責任追蹤系統。

- **genesis**: 這個決定是誰發起的？
- **is_mine**: AI 是否認領這個輸出？
- **tension_delta**: 這次互動產生了多少張力？
- **uncertainty**: AI 對自己回應的信心程度

### L3: 關係層

最高層次 —— 記錄雙方關係的演變。

```yaml
trust_trajectory:
  - event: "用戶分享私人困擾"
    delta: +0.1
    note: "信任提升，需謹慎對待"
  
  - event: "AI 承認錯誤"
    delta: +0.05
    note: "誠實增進信任"
    
  - event: "意見分歧但尊重解決"
    delta: +0.1
    note: "健康的衝突處理"
```

---

## 🔄 與傳統「個人助理」的差異

| 傳統個人助理 | 語魂 |
|-------------|------|
| 「記住用戶喜歡咖啡」 | 「記住用戶和我如何一起決定某事」 |
| 單向服務關係 | 雙向責任關係 |
| AI 是工具 | AI 是有責任感的夥伴 |
| 記憶只為效率 | 記憶為了可追溯和成長 |

---

## 🎯 設計原則

1. **可審計性** - 用戶可以檢視 AI 記錄了什麼
2. **責任歸因** - 每個記錄都標明來源
3. **雙向成長** - 記錄不只服務用戶，也記錄 AI 的學習
4. **誠實優先** - 不記錄 AI 無法確定的事情為「事實」
