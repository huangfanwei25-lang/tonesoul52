> Visual artifacts with **unresolved provenance / status**.

## 為什麼有這個資料夾

對應 Axiom 4（Non-Zero Tension）— 有些檔案的 epistemic state 真的是 unresolved、強行 categorize 進 `_archive/` 或 active 都是 over-claim。

但同時也對應 Axiom 1（Continuity）— 不能因為 unresolved 就讓 trace 消失（如把檔案留在 Temp 自然消亡）。

所以這個資料夾的存在本身、是 ToneSoul 對 "filing-with-honest-status" 原則的物質化：
- **檔案在 git 裡** ✓ 滿足 Continuity
- **路徑名 explicit 標 unresolved** ✓ 滿足 Non-Zero Tension
- **不混進 active 路徑** ✓ 不誤導 cold agent

## 何時離開這個資料夾

每個檔案應在 status 變更時 promote 或 archive：
- 確認 thesis-aligned → 移去 active 路徑（如 `apps/web/public/` 或 `docs/visual/` 主層）
- 確認 thesis-misaligned + 不再演變 → 移去 `_archive/`
- 朋友確認意圖（A 或 B 或 C）後、轉成上述其一
