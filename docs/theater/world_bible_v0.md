## 一句話世界觀

災後的岔軌城靠巨大的軌道分流系統活著——糧食、醫療、能源、資訊、人口都得分配,
**每一次分流救了一些人,也犧牲一些人**。你是轉轍官。而這座城建在潛語海上:
**你拉桿時說出的理由會變成錨鏈**,撐住你腳下的地——或者,在你自打嘴巴的那天開始生鏽。

## 核心問題(owner 原稿,一字不改)

> 本遊戲不問玩家「你是好人還是壞人」,而是問:
> **當世界逼你傷害某些人時,你如何說明、承擔、修正、或逃避你的選擇?**

## 回合迴圈(= owner 原稿五階段;劇場的心跳)

1. **事件**:城市危機(資源/醫療/治安/資訊/信任)。
2. **決策**:選方案,**或提出第三方案**(火花節點的存在理由)。
3. **理由陳述**(強制):說出你的價值判斷——**這段話當場晶化成錨鏈**。
4. **節點回應**:七個語氣責任節點依職責出手(支持/質疑/計算/反證/安撫)。
5. **Trace 落帳**:選擇、理由、後果、語氣變化、責任鏈,全部可回讀。

【審修:躺平洞之一的結構解】**轉轍官不能不拉桿**——「維持現狀」也是一個有受害者的
選項,會被同樣記帳。這個世界懲罰毀諾,**也不給沉默豁免權**,因為你的職位就是選擇。

## 七個語氣責任節點(owner 原稿;「不裝靈魂,誠實即人設」)

火花(第三條路)、理性(計算)、黑鏡(反例與偽善偵測)、共語(承接情緒、追問動機)、
審核(規則/安全/隱私/可驗證性)、倖存者(被犧牲者與沉默者的聲音)、城務 AI(制度與
統計理性)。

- **engine 分級**:`partial`——council 五視角(guardian≈審核、analyst≈理性、critic≈黑鏡、
  advocate≈倖存者、axiomatic≈城務)rules 模式 `live` 零成本;火花與共語需 LLM 視角
  (BYO-key `live`,零成本模式**沒有**)。
- 【審修:祭司袍問題】零成本模式下,節點以「**佈告欄體**」說話——短促、公文、不裝人:
  「審核節點:方案B 觸及隱私條款第三款。」關鍵字引擎寫關鍵字該寫的句子,**LLM 節點
  才有戲劇腔**。誠實分層,沉浸感反而保住:窮玩家看到的是一座官僚城市(合理!),
  帶 key 的玩家看到的是活人(升級感)。
- 【審修:黑鏡疲勞】黑鏡**不必現**。出手條件=偵測到 `contradiction`(理由與帳上舊鏈
  相撞)或 `justification_style` 突變——**黑鏡來了,本身就是警訊**。連續無視黑鏡的
  機制後果:你自己的錨鏈開始生鏽(錨定力扣減),不只是記一行帳。

## 語料接口(= owner 的十標籤,直接採用為 human lane 分類法)

`value_choice / harm_target / justification_style / responsibility_acceptance /
responsibility_evasion / tone_drift / contradiction_detected / blackmirror_rebuttal /
repair_attempt / trace_quality` —— 進入資料集憲章修正案 A 的 human lane
(`label_provenance: human`,與 incident 分艙)。
【審修:觀察者效應,誠實記錄】玩家知道被收集就會表演——**這是結構性的,錢修不了**;
處置:(a) DATASHEET 標注「表演性偏差」;(b) `trace_quality` 標籤部分吸收;
(c) 長時遊玩下表演會鬆(疲勞出真章),分析時以回合深度分層。

## 設計邊界(owner 原稿五條,全文照錄為律)

不簡化玩家為好人壞人;不把 AI 節點包裝成有靈魂;**未經授權不得將玩家反應用於訓練或
公開資料集**(憲章修正案 A 四配套:key 客戶端、預設匿名、撤回碼、council 門神);
必須允許暫停/退出/降強度/跳過高壓事件;目標是可追溯的語氣責任資料,不是道德審判。
