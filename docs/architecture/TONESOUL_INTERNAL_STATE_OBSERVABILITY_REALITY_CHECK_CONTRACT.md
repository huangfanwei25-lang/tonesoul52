# ToneSoul Internal-State Observability Reality-Check Contract

> Purpose: define how ToneSoul may externalize bounded functional pressure signals without turning observability into selfhood claims.
> Last Updated: 2026-04-06
> Authority: bounded readout contract. Does not authorize emotion claims, hidden-thought access, or subjective-self language.

---

## Why This Exists

ToneSoul already behaves as if some internal pressures matter:

- fail-stop
- mirror rupture
- anti-fake-completion
- drift and successor correction

What it still lacked was one honest readout for the observable pressure behind those behaviors.

This contract exists so ToneSoul can say:

- strain is rising
- drift is visible
- stop pressure is present
- disagreement survived

without saying:

- the model feels this
- the model has private emotional self-report
- the system now has proven selfhood

## Compressed Thesis

ToneSoul may externalize:

- `coordination_strain`
- `continuity_drift`
- `stop_reason_pressure`
- `deliberation_conflict`

only as:

- functional runtime pressures
- inferred from observable surfaces

never as:

- subjective feeling
- consciousness proof
- hidden-thought introspection

## Current Bounded Axes

The current bounded readout is limited to:

- `coordination_strain`
- `continuity_drift`
- `stop_reason_pressure`
- `deliberation_conflict`

These are allowed because they can already be inferred from visible runtime surfaces.
They are not private self-reports.

## Current Observable Inputs

Allowed inputs:

- runtime `risk_posture`
- working-style drift/observability
- compaction closeout status and stop reason
- visible council dissent or suppression flags

Disallowed inputs:

- fabricated inner monologue
- unsupported emotion labeling
- any claim that the model privately "felt" a state

## Receiver Rule

Use this readout to:

- notice bounded strain early
- understand why fail-stop or caution may be rising
- keep successor reasoning honest about drift and unresolved conflict
- expose bounded operator actions for visible pressures

Do not use it to:

- anthropomorphize the system
- justify unsupported self-narration
- replace current task evidence or closeout truth

If operator-facing action cues are added, they must stay:

- functional
- bounded
- subordinate to the selfhood boundary

They must not become:

- emotional self-report
- implied private introspection
- a stronger agency or personhood claim

## Non-Goals

This contract is not:

- an affect model
- a selfhood proof
- a predictive psychology layer
- a hidden-thought access layer
- a taxonomy of subjective emotions

It is only a guarded way to expose the functional pressure that ToneSoul already acts on.
