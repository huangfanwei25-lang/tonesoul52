# Status Artifacts

This folder is the repository status source for machine-generated governance snapshots.

## Files

- `7d_snapshot.json`
  - Generated by `scripts/run_monthly_consolidation.py`
  - Contains the latest structured output from `scripts/verify_7d.py`
- `monthly_consolidation_report.json`
  - Generated by `scripts/run_monthly_consolidation.py`
  - Aggregates:
    - 7D gate result
    - memory hygiene
    - architecture boundary contract
    - docs consistency checks
- `repo_healthcheck_latest.json`
  - Generated by `scripts/run_repo_healthcheck.py`
  - Aggregates:
    - python lint (`ruff`)
    - python format check (`black --check`)
    - python tests (`pytest tests -q`)
    - web lint (`npm --prefix apps/web run lint`)
    - web tests (`npm --prefix apps/web run test`)
    - 7D audit (`scripts/verify_7d.py`)
- `repo_healthcheck_latest.md`
  - Generated by `scripts/run_repo_healthcheck.py`
  - Human-readable summary of the latest healthcheck run
- `git_hygiene_latest.json`
  - Generated by `scripts/verify_git_hygiene.py`
  - Aggregates:
    - object-store snapshot (`git count-objects -vH`)
    - fsck diagnostics (`git fsck --no-reflogs`)
    - threshold evaluation (`max_loose_count`, `max_dangling`)
- `git_hygiene_latest.md`
  - Generated by `scripts/verify_git_hygiene.py`
  - Human-readable summary of the latest git hygiene snapshot

## Generation

```bash
python scripts/run_monthly_consolidation.py --strict
```

CI-friendly mode (when discussion files are absent in clean checkout):

```bash
python scripts/run_monthly_consolidation.py --strict --allow-missing-discussion
```

```bash
python scripts/run_repo_healthcheck.py --strict
```

CI-friendly mode (skip `verify_7d` when discussion file is absent):

```bash
python scripts/run_repo_healthcheck.py --strict --allow-missing-discussion
```

Live SDH mode-switch smoke (requires running web/backend services):

```bash
python scripts/run_repo_healthcheck.py --strict --include-sdh --check-council-modes
```

```bash
python scripts/verify_git_hygiene.py
```

Strict mode (optional, fail when thresholds are exceeded):

```bash
python scripts/verify_git_hygiene.py --strict --max-loose-count 5000 --max-dangling 50
```

## Automation

- GitHub Actions workflow: `.github/workflows/monthly_consolidation.yml`
- GitHub Actions workflow: `.github/workflows/repo_healthcheck.yml`
- GitHub Actions workflow: `.github/workflows/git_hygiene.yml`
- Trigger modes:
  - monthly schedule (`30 3 1 * *`, UTC)
  - weekly schedule (`15 3 * * 1`, UTC)
  - manual dispatch (`workflow_dispatch`)
- Workflow uploads `docs/status/*.json` as build artifacts for audit/review.

## Convergence Cadence

- Convergence week: freeze new framework additions, prioritize hygiene and consistency.
- Minimum maintenance loop:
  - `python scripts/run_repo_healthcheck.py --strict --allow-missing-discussion`
  - `python scripts/run_7d_isolated.py --web-script dev` (when SDH live evidence is needed)
  - `python scripts/verify_git_hygiene.py`

## Notes

- These artifacts are intended as operational truth for CI/review workflows.
- Human-facing narrative docs (`README.md`, sync notes) should reference these artifacts instead of duplicating volatile numbers.
