# Status Artifacts

This folder is the repository status source for machine-generated governance snapshots.

## How To Read This Folder

- Read markdown with UTF-8 in PowerShell:
  - `Get-Content -Raw -Encoding UTF8 docs/status/<file>.md`
- Treat `*_latest.json` and `*_latest.md` as generated point-in-time snapshots.
- Do not hand-edit generated `*_latest.*` artifacts unless you are fixing the generator itself.
- Separate machine snapshots from hand-authored reconciliation docs:
  - Generated operational snapshots:
    - `repo_healthcheck_latest.*`
    - `git_hygiene_latest.*`
    - `dual_track_boundary_latest.*`
    - `change_intent_latest.*`
    - `memory_*_latest.*`
    - `friction_shadow_*_latest.*`
    - `philosophical_reflection_latest.*`
  - Hand-authored historical / handoff docs:
    - `professional_ai_architecture_review_2026-03-04.md`
    - `professional_ai_architecture_positioning_2026-03-07.md`
    - `round2_progress.md`
    - `round2_handoff_framework.md`
    - `orphan_modules_report.md`
- If a generated snapshot conflicts with a historical review, prefer:
  1. commit/file evidence,
  2. generated latest snapshot for current state,
  3. hand-authored review for historical interpretation.

## Shared Compact Handoff Grammar

Several generated status artifacts now share a small cross-artifact contract so
upper mirrors can stay passive and avoid inventing their own summary language.

- Shared handoff metadata:
  - `path`
    - Optional artifact path for passive/focus preview mirrors that come from an existing generated status file.
  - `queue_shape`
    - The stable routing label used by preview selectors and focus-mirror choice.
  - `requires_operator_action`
    - The source-declared operator-action flag; mirrors should preserve it rather than infer it.
- Required compact lines:
  - `primary_status_line`
    - The one-line declaration of the artifact's current top-level posture.
- Common optional compact lines:
  - `runtime_status_line`
    - A bounded runtime or posture summary for the same artifact.
  - `scribe_status_line`
    - A Scribe-specific posture line when the artifact mirrors chronicle status.
  - `anchor_status_line`
    - The dominant lead anchor when an artifact exposes Scribe/state-document grounding.
  - `problem_route_status_line`
    - The dominant route-first diagnosis: failure family, broken invariant, and first repair surface.
  - `problem_route_secondary_labels`
    - Optional secondary route hints when one dominant family is not the whole story.
  - `dream_weekly_alignment_line`
    - A compact statement of whether weekly host-facing posture and dream observability agree on the dominant route family.
  - `semantic_retrieval_protocol`
    - An optional source-declared retrieval rule string for semantic-memory surfaces such as repo intelligence.
  - `semantic_preferred_neighborhood`
    - The preferred first semantic neighborhood to open before falling back to raw file search.
  - `artifact_policy_status_line`
    - A bounded policy line describing what kind of artifact/output was produced or mirrored.
  - `admissibility_primary_status_line`
    - Optional admissibility posture for authority/exception-pressure lanes.

When an artifact exposes these lines under its nested `handoff` object, upper
preview surfaces should preserve the source-declared line instead of silently
recomputing a different wording. Recalculation is a fallback only when the
source artifact did not declare the compact line itself.

## Automation

- GitHub Actions workflow: `.github/workflows/monthly_consolidation.yml`
- GitHub Actions workflow: `.github/workflows/repo_healthcheck.yml`
- GitHub Actions workflow: `.github/workflows/git_hygiene.yml`
- GitHub Actions workflow: `.github/workflows/persona_swarm.yml`
- GitHub Actions workflow: `.github/workflows/external_source_registry.yml`
- GitHub Actions workflow: `.github/workflows/friction_shadow_calibration.yml`
- GitHub Actions workflow: `.github/workflows/dual_track_boundary.yml`
- Trigger modes:
  - monthly schedule (`30 3 1 * *`, UTC)
  - weekly schedule (`15 3 * * 1`, UTC)
  - manual dispatch (`workflow_dispatch`)
- Repo healthcheck manual dispatch supports SDH inputs:
  - `include_sdh` (boolean)
  - `web_base`, `api_base`, `sdh_timeout`
  - `check_council_modes` (boolean)
  - Dispatch orchestration script: `scripts/run_repo_healthcheck_dispatch.py`
  - Validation behavior:
    - invalid `sdh_timeout` (non-positive integer) blocks run
    - SDH inputs with `include_sdh=false` emit warning and are ignored
    - only one of `web_base` / `api_base` emits warning; missing side falls back to `verify_7d` default
- Persona swarm workflow dispatch:
  - inputs: `strict`, `input_path`
  - dispatch orchestration script: `scripts/run_persona_swarm_dispatch.py`
  - validation behavior:
    - missing `input_path` blocks run with workflow error annotation
    - non-`.json` input path emits warning but still runs
- Workflow uploads selected `docs/status/*.json`, `*.md`, and `*.jsonl` artifacts for audit/review.
