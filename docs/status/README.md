# Status Artifacts

This folder is the repository status source for machine-generated governance snapshots.

## Files

- `7d_snapshot.json`
  - Generated by `scripts/run_monthly_consolidation.py`
  - Contains the latest structured output from `scripts/verify_7d.py`
- `monthly_consolidation_report.json`
  - Generated by `scripts/run_monthly_consolidation.py`
  - Aggregates:
    - 7D gate result
    - memory hygiene
    - architecture boundary contract
    - docs consistency checks
    - persona swarm readiness check
- `repo_healthcheck_latest.json`
  - Generated by `scripts/run_repo_healthcheck.py`
  - Aggregates:
    - python lint (`ruff`)
    - python format check (`black --check`)
    - python tests (`pytest tests -q`)
    - web lint (`npm --prefix apps/web run lint`)
    - web tests (`npm --prefix apps/web run test`)
    - git hygiene (`scripts/verify_git_hygiene.py`)
    - dual-track boundary (`scripts/verify_dual_track_boundary.py --strict --staged`)
    - persona swarm readiness (`scripts/run_persona_swarm_framework.py --strict`)
    - 7D audit (`scripts/verify_7d.py`)
- `repo_healthcheck_latest.md`
  - Generated by `scripts/run_repo_healthcheck.py`
  - Human-readable summary of the latest healthcheck run
- `git_hygiene_latest.json`
  - Generated by `scripts/verify_git_hygiene.py`
  - Aggregates:
    - object-store snapshot (`git count-objects -vH`)
    - fsck diagnostics (`git fsck --no-reflogs`)
    - threshold evaluation (`max_loose_count`, `max_dangling`)
- `git_hygiene_latest.md`
  - Generated by `scripts/verify_git_hygiene.py`
  - Human-readable summary of the latest git hygiene snapshot
- `dual_track_boundary_latest.json`
  - Generated by `scripts/verify_dual_track_boundary.py`
  - Captures dual-repo boundary policy results for changed paths
  - Includes:
    - blocked prefix/file policy snapshot
    - changed file source metadata
    - violation list with matched policy rules
- `dual_track_boundary_latest.md`
  - Generated by `scripts/verify_dual_track_boundary.py`
  - Human-readable summary of latest dual-track boundary check
- `multi_persona_eval_latest.json`
  - Generated by multi-persona evaluation runs (Phase 78 protocol)
  - Tracks A/B/C comparison metrics and promotion decision snapshot
- `persona_swarm_framework_latest.json`
  - Generated by persona swarm framework validation runs (`scripts/run_persona_swarm_framework.py`)
  - Captures swarm coordination metrics, gate decisions, persona readiness diagnostics
  - Includes governance signals:
    - guardian fail-fast trigger state
    - cost profile tier and recommended agent budget
  - Includes runtime budget application fields:
    - `baseline_evaluation` (full signal pass)
    - `execution_plan` (selected agents after budget policy)
- `external_source_registry_latest.json`
  - Generated by `scripts/run_external_source_registry_check.py`
  - Captures external source allowlist validation:
    - `https` scheme enforcement
    - blocked short-link host rejection
    - allowlist host enforcement
    - source `reviewed_at` freshness checks
    - `spec/open_source_apps.yaml` repo/website URL compliance
- `external_source_registry_latest.md`
  - Generated by `scripts/run_external_source_registry_check.py`
  - Human-readable summary of latest external source registry validation

## Automation

- GitHub Actions workflow: `.github/workflows/monthly_consolidation.yml`
- GitHub Actions workflow: `.github/workflows/repo_healthcheck.yml`
- GitHub Actions workflow: `.github/workflows/git_hygiene.yml`
- GitHub Actions workflow: `.github/workflows/persona_swarm.yml`
- GitHub Actions workflow: `.github/workflows/external_source_registry.yml`
- GitHub Actions workflow: `.github/workflows/dual_track_boundary.yml`
- Trigger modes:
  - monthly schedule (`30 3 1 * *`, UTC)
  - weekly schedule (`15 3 * * 1`, UTC)
  - manual dispatch (`workflow_dispatch`)
- Repo healthcheck manual dispatch supports SDH inputs:
  - `include_sdh` (boolean)
  - `web_base`, `api_base`, `sdh_timeout`
  - `check_council_modes` (boolean)
  - Dispatch orchestration script: `scripts/run_repo_healthcheck_dispatch.py`
  - Validation behavior:
    - invalid `sdh_timeout` (non-positive integer) blocks run
    - SDH inputs with `include_sdh=false` emit warning and are ignored
    - only one of `web_base` / `api_base` emits warning; missing side falls back to `verify_7d` default
- Persona swarm workflow dispatch:
  - inputs: `strict`, `input_path`
  - dispatch orchestration script: `scripts/run_persona_swarm_dispatch.py`
  - validation behavior:
    - missing `input_path` blocks run with workflow error annotation
    - non-`.json` input path emits warning but still runs
- Workflow uploads `docs/status/*.json` as build artifacts for audit/review.

## Convergence Cadence

- Convergence week: freeze new framework additions, prioritize hygiene and consistency.
- Minimum maintenance loop:
  - `python scripts/run_repo_healthcheck.py --strict --allow-missing-discussion`
  - `python scripts/run_7d_isolated.py --web-script dev` (when SDH live evidence is needed)
  - `python scripts/verify_git_hygiene.py`
