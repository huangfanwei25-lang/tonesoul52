# ToneSoul Outreach Platform Notes

Purpose: keep outreach drafts useful to humans and aligned with platform norms. This is not a bypass guide and does not guarantee acceptance.

## Policy posture

- Generate drafts, not automated posts.
- Preserve platform-specific context instead of posting the same text everywhere.
- Add AI-use disclosure when the platform expects it or when the draft is longform.
- Keep an evidence boundary in longform drafts: public architecture and conceptual prototype, not safety proof, sentience claim, or production guarantee.
- Block or mark as manual-only when a platform's rules are strict.

## Platform summary

| Platform | Generator posture | Reason |
| --- | --- | --- |
| LessWrong | Manual review | Best fit for explicit AI alignment framing and falsifiable objections. |
| AI Alignment Forum | Manual review | Non-members face review or LessWrong promotion path. |
| EA Forum | Manual review | Crosspost/linkpost is acceptable when summary and relevance are clear. |
| Hugging Face Blog | Manual review | Requires high-quality, long, technical, original ML posts. |
| DEV Community | Manual review | AI-assisted writing is allowed with disclosure and fact checking. |
| Substack | Manual review | Good canonical home, but not for SEO-only or spammy marketing. |
| Hacker News | Manual-only packet | Submit original source only; do not paste generated comments. |
| Reddit | Manual review | Rules are subreddit-specific; avoid duplicated self-promotion. |
| Bluesky / Mastodon / X | Draft only | APIs may exist, but repeated or substantially similar automated posts are risky. |
| LinkedIn | Manual only | LinkedIn API terms prohibit using APIs to automate posting. |
| arXiv | Blocked without peer review | CS review/position papers require accepted peer review documentation. |
