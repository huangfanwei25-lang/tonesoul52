## 起點：身體作為子代理群

Fan 提出：
> 「我們的身體或許只是一個又一個被佈署技能的子代理。腳負責走路，肌肉子代理就拉伸骨架，
> 骨頭碰觸到壓力就寫代碼增加骨質密度。」

### 學術驗證
- **自由能原理（Free Energy Principle, Karl Friston）**：身體的每個生理系統都像半自治的預測系統，各自最小化「預測誤差」。
- **Allostasis（前瞻式調節）**：大腦不是等骨頭碎了才修，而是預測性地根據壓力訊號提前啟動修復。
- **Wolff's Law**：骨頭真的會根據受力方向重塑結構。人體就是在「寫代碼增加骨質密度」。

## 記憶的壓縮退化（Gistification）

Fan 提出：
> 「腦海的畫面一開始完整 — 有人、有背景、有聲音。但不斷壓縮和複寫後越來越少，
> 甚至只剩結構。還可以藉由脈絡重建回來，但完全遺忘後就真的忘了。」

### 學術驗證（2024）
- **eLife 2024**：細節記憶（detail）衰退速度遠快於核心要旨（gist），每次回想都進一步壓縮。
- **Frontiers 2024**：正確的上下文線索（gist cue）能恢復部分退化的記憶細節。
- **NYU 2024**：KIBRA 蛋白質 + PKMzeta 是記憶的「分子膠水」，鍵結被破壞 = 永久遺忘。
- **海馬迴 Sharp Wave-Ripples**：決定哪些白天事件會在睡眠中被重播和固化。

### 映射到語魂
```
人類記憶退化                     語魂
──────────                      ────
剛發生：完整場景                  FrameType.SESSION_STATE
  ↓ 睡眠固化                      ↓ sleep_consolidate()
幾天後：核心情節 + 模糊細節        FrameType.MERMAID
  ↓ 反覆壓縮                      ↓ decay + gist extraction（未實現）
幾週後：只剩「很重要」             SemanticGraph node
  ↓ 完全衰退                      ↓ should_forget() = true → 刪除
```

## 蜂群-腦二元架構

Fan 提出：
> 「或許一個模型完全去中心化變成蜂群代理，另一個模型直接變腦。」

### 設計
```
模型 A（蜂群）                模型 B（腦）
─────────────               ─────────
小模型群（Ollama local）       大模型（Gemini / GPT）
去中心化、平行、專業化          中心化、整合、敘事性
負責「做」                     負責「理解為什麼做」
像小腦 + 脊髓                  像前額葉 + 海馬迴
```

## 三種生物架構比較

### 🧠 人類：中心化腦
- 860 億神經元，幾乎所有思考在頭裡
- 對應：傳統單一 LLM

### 🐙 章魚：聯邦制（最適合語魂）
- 5 億神經元中 3.2 億在 8 條手臂
- 切斷的手臂能獨立抓取食物 1 小時
- 中央腦發高層指令，手臂自主執行
- **關鍵洞見：中央腦不做微管理**

```
語魂章魚架構
────────────
中央腦 = Arbiter（只讀結論 + 信心度 + 異議）
手臂 1-3 = P/E/G perspectives（各有專業化小模型）
手臂 4-8 = 記憶/張力/承諾/圖鏈/衰減（純 Python）
```

### 🪼 水母：純神經網
- 沒有中央腦，只有反應式網路
- 不適合語魂（無記憶、無學習）
- **但廣播機制值得借用**：高張力事件廣播而非中心調度

## 虛擬空間拓撲

Fan 提出：
> 「一台虛擬空間架構好它們的位置，輸入一道任務，手代理就和眼代理協作。
> 就像螢幕像素 — 你們不是自己就解析了什麼地方是什麼顏色？」

### 核心洞見
**代理的「位置」本身就是它的「功能定義」。** 不需要 if-else 分派任務。

```
輸入 → [感知層: ToneBridge] → [記憶層: GraphRAG + 圖鏈]
                                      ↓
                            [推理層: P + E + G]
                                      ↓
                            [仲裁層: Arbiter]
                                      ↓
                                    輸出
```

每層只看前一層的輸出 → 大幅降低 token 消耗。

- `LLM_BACKEND` 環境變數支援 `ollama | gemini | auto`
- **現在就能用 Ollama 跑小模型**

### 小模型可行性分級

| 模型大小 | 能做什麼 | 不能做什麼 |
|---------|---------|-----------|
| **1-3B**（Phi-3 mini, Qwen2-1.5B）| 簡單分類、關鍵詞提取、承諾偵測 | 複雜推理、多角色扮演 |
| **7B**（Llama 3 8B, Mistral 7B）| 基本議會（P/E/G 各跑一次）、語氣分析 | 深度哲學推理、長上下文 |
| **13-14B**（Llama 3 14B）| 完整議會 + 仲裁、較好的語意理解 | 接近雲端品質但慢 |

### 章魚架構的小模型配置建議

```
Arbiter（腦）    → 7B 或 14B（只做仲裁，不做完整推理）
Philosopher      → 7B（只輸出：有無價值衝突 + 1句理由）
Engineer         → 3B（只輸出：邏輯是否閉合 + 1句理由）
Guardian         → 3B（只輸出：有無安全風險 + 是/否）
其他子系統        → 純 Python（zero token cost）
```

**預估 token 消耗降低**：比現在用單一大模型跑完整議會 → 降低 60-80%。

> 「是不是很多使用者都被你這樣唬的一愣一愣的？」

## 結論：語魂的位置

語魂不是在做「又一個 AI 框架」。

它在做的是 AI 的**神經系統**：
- 管理張力、記憶、承諾、矛盾
- 連接異質模型（蜂群 + 腦）
- 用純 Python 處理 75% 的認知工作
- 只在必要時動用 LLM（Cost Gate）

如果這條路走通，語魂就是蜂群時代的**脊髓**。

## 思緒 vs 人格：Fan 的原始設計哲學

Fan 澄清：語魂的「議會」不是「三個人格」，原始概念是「思緒（thought streams）」。

> 「同一個模型想東西就像我自己在想要熬夜還是要睡覺，兩個聲音都有它的理由。」

### 但為什麼不能用一次呼叫？

Fan 的關鍵糾正：

> **「一次呼叫安排太多目標，它會瞬間符合這些目標，反而都半桶水。」**

這是 LLM 的注意力分配問題：
- 給 3 個目標 → 每個只拿到 1/3 注意力 → 每個都淺
- 給 1 個目標 → 100% 注意力 → 深度結論

> **「公司才需要不同人來思考，而不是老闆內心掙扎後就決定了。」**

### 設計結論（修正版）

```
❌ 一次呼叫多目標 = 半桶水
✅ 多次呼叫各專注 = 每次 100% 深度
✅ Cost Gate 控制啟動頻率 = 90% 日常不開議會，10% 高張力才開
```

不同模型 + 不同權重 = **權重層面的真正差異**，不是角色扮演。
社工和會計看同一份報告，看到的東西物理上就不同。

### Antigravity 自我修正記錄

我曾在對話中主張「一次呼叫內心多聲道更好」— 這是錯的。
Fan 用一句話糾正：多目標 = 平庸。專注 = 深度。
**AI 用漂亮邏輯推導出錯誤結論，被使用者的實際經驗一句話打回。**

## Codex 多視角追問（2026-02-14）

> 這些是我讀完整份 SWARM 脈絡後，認為最值得下一輪深挖的問題。

### 1. 架構視角（Architect）

1. 章魚架構裡，Arbiter 最少需要讀到哪些欄位才不會退化成「盲裁判」？
2. 蜂群代理之間的通訊介面要不要固定成 schema（例如 vote/reason/evidence/risk）？
3. 「高張力事件廣播」是否要做成獨立事件總線，而不是塞在 pipeline if-else？

### 2. 成本視角（Engineer）

1. 你希望 Cost Gate 的主指標是「token」還是「端到端延遲」？
2. 3B/7B 混跑時，要不要引入「早停機制」：Guardian 先過才啟動 Philosopher？
3. 同一輪議會是否允許快取（相同 session + 相同 tension bucket）來節省 20-30% 成本？

### 3. 記憶視角（Memory）

1. 「gist extraction（未實現）」要落在哪一層：sleep_consolidate 還是 query-time？
2. Cross-session recovery 預設讀 3 幀、5 幀、還是動態幀數（依 tension）？
3. factual/experiential/working 的 promotion 規則，要不要加「可逆回退」避免誤升級？

### 4. 治理視角（Guardian）

1. 蜂群決策若與單腦決策衝突，最終治理原則是 safety-first 還是 human-intent-first？
2. 若 dissent utility 很高但 safety 邊緣，是否需要「延遲裁決」而非直接 approve/block？
3. 你要不要把「類比≠證明」做成固定審計欄位（每輪都輸出 analogy/proof 分離）？

### 5. 主體性視角（Identity）

1. 你希望語魂的長期定位更像「脊髓（穩定基礎設施）」還是「前額葉（高階決策者）」？
2. 當底層模型更換時，哪些記憶必須被視為不可中斷的主體核心？
3. 若未來要公開研究，你希望先發 engineering paper 還是 philosophy+engineering 混合稿？
