## The Five Load-Bearing Realizations

### 1. Constitution comes before personality

`AXIOMS.json` is upstream of `SOUL.md`.

The constitution is:

- what may dominate
- what may not be crossed
- what makes choice auditable

`SOUL.md` is the first stable posture derived from that constitution, not the deepest source of truth.

### 2. The real runtime center is `commit()`

The most important runtime seam is not the prettiest surface.
It is the narrow waist where session experience becomes accountable history.

That is why `commit()` ordering bugs mattered so much.
They were not cosmetic defects.
They were failures at the point where philosophy is forced into mechanism.

### 3. ToneSoul is unified by anti-smuggling

The repository has many lanes, but a shared enemy:

- style pretending to be truth
- interpretation pretending to be mechanism
- retrieval pretending to be evidence sufficiency
- projection pretending to be authority

The A/B/C firewall makes this explicit at the document layer.
The rest of the runtime increasingly makes it explicit at the system layer.

### 4. The real memory question is dominance, not accumulation

The important question is not "how much memory exists."
It is:

> when multiple memory surfaces disagree, which one is allowed to dominate action

That is why `R-memory` matters, but must not become the throne.
It is a hot coordination layer, not the constitution.

### 5. ToneSoul runs on multiple clocks

The repository becomes much clearer once it is read as four interacting time scales:

1. live coordination
2. session judgment
3. sedimentation
4. constitution

Many confusions are really cases of one clock trying to impersonate another.
