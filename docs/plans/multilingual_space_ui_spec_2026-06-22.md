# Spec — Multilingual try-it Space UI (work order for Codex)

> Status: bounded work-order spec; **not** ratified into task.md (ratification is an owner act).
> Author: Claude (Opus 4.8), architect lane · Implementer: Codex · Reviewer: Claude (cross-AI loop).
> Last updated: 2026-06-22.
>
> **Framing — read first.** This is **distribution, not capability.** Goal: localize the try-it
> Space's UI **chrome** so a non-English visitor — especially the Chinese-reader audience the
> project targets — can navigate it in their language. It adds **no governance capability**, changes
> **no engine**, and is honestly scoped: the analysis **output stays English** (it is
> engine-generated); **only the chrome is localized**. Do not let it imply "multilingual analysis".
> The funnel it completes: a Chinese-language post → a Chinese-labelled try-it → (English analysis,
> honestly flagged).

## Context

`apps/try/app.py` is a Gradio Space (gradio **6.19.0** on HF — see `apps/try/README.md`
`sdk_version`). It wraps the model-free `PreOutputCouncil`. The live Space is
`Famwin/tonesoul-tryit` — a **separate HF repo, NOT git-connected** to GitHub. So this work delivers
an updated `apps/try/app.py` in this repo, and the **human owner re-uploads it to the Space**. Do
**not** attempt to deploy or touch the Space.

## Goal

Localize the UI chrome using Gradio 6's **built-in** i18n (`gr.I18n` — **verify the exact API
against gradio 6.19.0**; it differs from 5.x). Gradio auto-selects by browser locale; `en` is the
default/fallback. Locales: **en, zh-TW, zh-CN, ja** (trim `ja` to just zh + en if it adds noise —
your call, state it).

Strings to translate (chrome only):
- the title (`🦞 ToneSoul — try it on your own output`),
- the two input labels (`Your draft output` / `Optional — the user intent it responds to`),
- the `DESCRIPTION` block,
- the empty-input message (`Paste a draft answer first.`),
- the static section headers emitted by `run_council` **if** cleanly separable (Verdict / Coherence
  / Per-perspective) — only the **static labels**, never the engine-generated reasoning text.

## Honest scope / limits (must hold)

- The analysis **output** (verdict, per-perspective reasoning, evidence branches) is generated by
  the engine **in English** and is **not** translated. In each non-English locale, the
  `DESCRIPTION` must carry **one honest line** stating the analysis output is in English.
- **Keep `DEFAULT_DRAFT` in English**: the gates are lexically English-oriented, so a localized
  example would not trigger the same flags and would misrepresent behaviour. A short localized
  caption around it is fine; the draft text stays English.
- **No engine change**, model-free, deterministic. **No new dependency** (i18n is built into
  gradio). **No new capability** — chrome localization only.

## Verification (run what is runnable; be honest about the gap)

- `ruff` + `black --check` on `apps/try/app.py`.
- `python -m py_compile apps/try/app.py` (syntax). Note: gradio may not be importable in the repo
  env — if import fails **only** because gradio is missing, that is expected; say so explicitly,
  do not pretend the UI ran.
- **Structural self-check:** every i18n key used has a value in **every** locale (no missing-key
  fallback surprises). Include the key × locale matrix in your report.
- **Final functional verification is on the live Space** after the human re-uploads. State this;
  do **not** claim the UI was verified locally if gradio could not run.

## Non-goals (do NOT do)

- Do **not** translate the engine output or claim multilingual analysis.
- Do **not** change `PreOutputCouncil` or anything under `tonesoul/`.
- Do **not** auto-deploy or touch the HF Space (separate repo; the human re-uploads).
- Do **not** add `gradio` to `requirements.txt` (pre-installed on a Gradio Space).

## Hand-off

Codex implements `apps/try/app.py`; Claude reviews (ruff/black/syntax + the i18n key-coverage matrix
+ the honest-scope check — engine output stays English, example stays English) and lands; the human
owner re-uploads `app.py` to the Space. Local verification is bounded (no gradio runtime); the live
Space is the real check.
