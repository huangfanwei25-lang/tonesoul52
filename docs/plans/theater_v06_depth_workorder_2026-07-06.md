# 劇場 v0.6 深度化 — 工單(2026-07-06,owner 首次實玩回饋)

> 性質:**大改版總工單**,拆給不同引擎/owner;每項標 lane + 是否需 owner 先拍。
> owner 授權形狀:「有把握的你來做(=V05-E 已做),太大的寫成工單交給其他模型 OPUS/CODEX」。
> 本檔=那些「太大的」。設計參考研究另存 session 記錄(Papers Please / Obra Dinn / Reigns /
> VA-11 / 80 Days / Disco Elysium / Pentiment / Frostpunk);核心洞見引在下。
> **克制論旨守則(每項都受它約束)**:這是問責/記錄器,不是能力/最佳化遊戲。任何讓玩家
> min-max、產生「正解」、把反思變成被計時懲罰的雜務的設計——拒絕。

## 核心設計洞見(研究結論,最高槓桿)

**「記錄存在的理由 = 遊戲存在的理由」**:曾有人大規模改寫所有人的記憶,唯一防線是一條
不可竄改、自己說過的話的鏈——這正是黑鏡機制在做的事。**世界觀(e)、主題(c)、
機制三者是同一件事**:黑鏡引用你自相矛盾的那一刻,與造成浩劫的動作(抹掉說過的話)
同構。這條線把「為什麼要記錄決策」從說教變成玩法本身。

**Disco Elysium 五聲音 = 語魂五視角議會**:每站讓 5 個聲音各說一句短台詞,玩家「慣於
聽哪個聲音」就顯露性格——**一次同時解 (b)角色 (c)主題 (e)世界觀**。這是本次最該先做的。

## V06-A 站上議會五聲音(最高優先;解 b/c 大半)

- **痛點**:(a)劇情太短、(b)角色特色交代不清、(c)每站想讓玩家思考的主題沒浮現。
- **形狀**:每站決策前,5 個議會視角(Guardian/Analyst/Critic/Advocate/Axiomatic)各出
  1 句該站主題相關的短台詞(Disco 式內閣)。該站主題 = 哪個聲音領銜(記憶水庫=誠實vs慈悲、
  治安軌道=秩序vs個體…)。玩家常聽哪個聲音 → 顯露性格,零額外長文。
- **三行夾住主題**(省字技術):①站銘(入站一句公案,如記憶水庫「水記得每一次開閘」)→
  ②議會主聲領銜 → ③收尾刺針(出站一句點名代價:「你選了秩序。第 3 排的名字不會再被念到」)。
- **Lane 拆分**:
  - *機制*(codex,bounded):gamedata 加 `station_voices`/`station_epigraph`/`closing_sting`
    欄位,app.js 渲染;缺欄 fallback 現狀。驗收:有欄顯示、無欄不壞。
  - *內容*(**owner 或敘事 opus session**,非 codex):寫 15 站 × 5 聲音台詞 + 站銘 + 刺針。
    這是角色與世界的靈魂,codex 不擅長;建議 owner 定調 1-2 站當範本,再由 opus 續寫、owner 審。
- **owner 先拍**:五聲音要不要用真 council 引擎跑(離線重播)還是純敘事腳本。建議純敘事
  (引擎跑會過重;火花節點仍 absent 標注)。

## V06-B 世界觀:記憶改寫浩劫(解 e;owner canon)

- **形狀**(研究:漸進教學,不倒 lore):浩劫從玩家已在觸碰的物件滲出——情報單抬頭/戳記
  提「大改寫」不解釋;記憶法庭/水庫的規章暗示歷史(「凡裁決必留原話——第 X 次改寫後立」);
  過站解鎖一句「紀錄殘片」進 anchor chain,亂序給、意義自己浮現(Obra Dinn 式)。完整浩劫
  留到近結尾,且與黑鏡揭露矛盾的那刻扣合。**每站至多 1-2 句世界,永遠附在玩家正用的東西上。**
- **Lane**:**canon = owner**(這是你的世界,誰寫立法權在你)。可選:owner 定浩劫骨架
  (三五句),敘事 opus 依骨架把殘片鋪進 15 站,owner 終審。**codex 不碰 canon**。
- **接線機制**(codex,骨架定案後):`intel.lore_fragment` 欄位 + 過站滴餵 + anchor chain 顯示。

## V06-C 經營玩法(解 d;**owner 必先拍——thesis 風險**)

- **這是 DECLARE_STANCE:攤兩案,選擇權在你,不替你決。**
- **推薦形狀(服務問責)**:3 格對立稀缺資源(秩序/信任/記憶容量),每次扳軌把它們往
  相反方向推、不能刷回填;每次顯示「這格 −1 = 第幾排的誰」(具名受害者);結局把三格
  形狀寫進 responsibility-trace,無勝負。沉默與第三路一樣付費。原型:Reigns 錶 + Frostpunk
  希望/不滿 + This War of Mine 的罪咎。
- **拒絕形狀(能力賽道,thesis 明拒)**:城市建造經濟迴圈、升級吞吐、為高分最佳化車流、
  科技樹、效率指標——招 min-max、製造「正解」,與「不判善惡、無最佳解」衝突。也拒絕:
  把寫 REASON 變成被計時懲罰的雜務。
- **與現況關係**:三資源(醫療/能源/信任)已存在但偏靜態。推薦形狀 = 讓它們變成「對立、
  不可回填、具名受害者」,不是新增經濟系統。**owner 拍板後**,機制 lane 可派 codex。

## V06-D 選地 + 時間進程(解 g;owner 設計拍板)

- **形狀**(80 Days 式):玩家在站點網路自選順序,時間是不可逆壓力(非成長時鐘),一輪
  走不完全部——「選擇不去哪」本身是一筆被記錄的省略(接問責主題)。
- **改動範圍**:現在是線性 playlist(1-7 線性、8 岔口、9-11 線性)。改自選會動 buildPlaylist
  與推進感 UI,中大型。**owner 先拍**:要不要打破線性(線性有敘事累積的好處,自選有能動性
  但可能散)。建議折衷:保留主線骨架,開放 2-3 站的順序 + 明確標「你略過了 X」。
- **Lane**:owner 拍方向 → codex 實作 playlist/UI(bounded)。

## V06-E 立繪登場 splash(解 f;小,codex)

- **形狀**:NPC 首次登場時,一張置中 splash(現有 bust 立繪放大 + 名字 + 一句身分),
  淡入後進對話。owner 若要「全身立繪」可後續換圖,splash 邏輯先上。
- **Lane**:codex(bounded,小);gamedata 記「首次登場」或 app.js 追蹤已見 NPC 集合。
- 誠實限制(研究):立繪只解決登場印象,解不了「性格不可展現」——必須配 V06-A 對白,
  否則畫再好仍是空殼。**V06-E 不可先於 V06-A 被當成 (b) 的解答。**
