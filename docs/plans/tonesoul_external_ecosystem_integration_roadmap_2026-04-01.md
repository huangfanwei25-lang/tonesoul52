## Bucket 1: Outcome-Calibrated Council Confidence

Source inspiration:
- TradeMemory outcome-weighted recall
- Bayesian-style confidence accumulation

Why this matters:
- directly addresses ToneSoul's current limitation that council confidence is descriptive rather than calibrated

### Candidate phases

1. Add Bayesian `alpha` / `beta` fields to council verdict carry surfaces without changing runtime behavior yet.
2. Add an outcome-feedback surface for later confirmation / override signals.
3. Surface descriptive confidence and Bayesian posterior side by side in later readouts.

### Why it stays deferred for now

This is a strong future bucket, but it is not cleaner than the current observer-window / collaborator-beta validation short board unless validation stalls.

## Bucket 2: Power-Law Tension Decay

Source inspiration:
- ACT-R style recall / decay patterns
- TradeMemory recall weighting

Why this matters:
- may improve Soul Integral retention behavior beyond the current pure exponential decay

### Candidate phases

1. Add a configurable power-law decay option alongside the current exponential model.
2. Add bounded retrieval-boost effects when tension events are repeatedly recalled.

### Why it stays deferred for now

Interesting, but still optimization. It should not outrun validation of the current continuity stack.

## Bucket 3: MCP Transport Adapter

Source inspiration:
- OpenMemory manager / MCP transport pattern

Why this matters:
- would make ToneSoul memory/governance surfaces discoverable to MCP-native tools without replacing current gateway logic

### Candidate phases

1. Add a thin MCP transport wrapper around a small subset of gateway surfaces.
2. Add bounded lifecycle / reference counting only after the transport surface proves useful.

### Why it stays deferred for now

Useful outer-shell work, but launch-default coordination is still file-backed and current collaborator-beta posture does not depend on MCP.

## Bucket 4: Prompt Assembly Architecture

Source inspiration:
- AI Agent Deep Dive v2
- prompt runtime assembly, fast-path entry, hook/permission layering

Why this matters:
- supports ToneSoul's current direction of bounded prompt discipline and entry/readout layering

### Candidate phases

1. Split more prompts into static role prefix + dynamic state suffix.
2. Add a pre-council snapshot hook if council-state capture becomes the shortest board.

### Why it stays deferred for now

ToneSoul already has prompt-discipline adoption in several generic surfaces. This should only resume when a real prompt/readout gap becomes shortest again.
