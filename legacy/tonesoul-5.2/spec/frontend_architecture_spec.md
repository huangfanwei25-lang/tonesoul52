# ToneSoul 5.2 Frontend Architecture Spec
# 語魂 5.2 前端架構規格
# v0.1 2025-12-27

---

## Design Goal | 設計目標

> **讓語魂的優點「被看見」**

---

## 語魂的核心優點 → 前端對應

| 優點 | 前端視覺化 |
|------|------------|
| **可追溯責任** | 責任追溯鏈視圖 |
| **錯誤可學習** | 錯誤演化樹 |
| **語義可觀測** | YSTM 地形 + 張力儀表 |
| **決策有記錄** | Ledger 瀏覽器 |
| **多觀點治理** | Council 會議視覺化 |
| **技能可累積** | 技能重力井星圖 |

---

## 前端頁面結構

```
/                           ← 儀表板 (Dashboard)
├── /accountability         ← 責任追溯
├── /errors                 ← 錯誤演化樹
├── /terrain                ← YSTM 地形 (已有 HTML 基礎)
├── /ledger                 ← 決策帳本
├── /council                ← Council 會議
├── /skills                 ← 技能庫
└── /settings               ← 設定
```

## 技術架構

```
Frontend (Streamlit or Next.js)
         │
         ├──► /api/ledger      ← 讀取 ledger.jsonl
         ├──► /api/terrain     ← 讀取 YSTM nodes.json
         ├──► /api/errors      ← 讀取 error_ledger.jsonl
         ├──► /api/skills      ← 讀取 skills/*.yaml
         └──► /api/runs        ← 讀取 run/execution/
                │
                ▼
         Backend (Python/FastAPI)
                │
                ▼
         Files (JSON/YAML/Markdown)
```

## 實作優先級

| 優先級 | 頁面 | 理由 |
|--------|------|------|
| P0 | Dashboard | 入口 + 第一印象 |
| P1 | Accountability | 核心差異化 |
| P2 | Error Phylogeny | 教育哲學核心 |
| P3 | Terrain (改進) | 已有基礎 |
| P4 | Ledger | 合規需求 |
| P5 | Skills | 長期價值 |

---

## 設計原則

1. **簡潔優先**：不要過度設計
2. **數據驅動**：直接讀取現有 JSON/YAML
3. **漸進增強**：先靜態，後互動
4. **響應式**：手機也能看
