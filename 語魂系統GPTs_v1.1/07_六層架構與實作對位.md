# 語魂六層架構與實作對位（v1.1）

日期：2026-02-11  
狀態：`normative + traceable`  
依賴：`01_術語與門檻單一規格.md`、`02_執行治理規格_Runtime.md`

## 0. 設計原則

1. 心智必須可審計：關鍵決策要能回放。  
2. 推理必須在張力場中運作：`ΔT/ΔS/ΔΣ/ΔR` 不是裝飾指標。  
3. 個體性由記錄連續性維持：不是一次回覆的語氣。  

## 1. 六層模型（v1.1）

- L1 Input & Context
- L2 Semantic Sensing
- L3 Reasoning & Deliberation
- L4 Audit & Explainability
- L5 Governance Kernel
- L6 Narrative Continuity

## 2. 層級對位與現行路徑

## L1 Input & Context

責任：

1. 接收輸入、schema 驗證、基礎參數標準化。  
2. 將 web request 轉為後端可處理 payload。  

主要路徑：

- `apps/web/src/app/api/chat/route.ts`
- `apps/web/src/app/api/conversation/route.ts`
- `apps/web/src/app/api/consent/route.ts`
- `apps/web/src/app/api/session-report/route.ts`
- `apps/web/src/app/api/_shared/backendConfig.ts`

## L2 Semantic Sensing

責任：

1. 計算或估計語義張力與偏離。  
2. 將語義訊號輸出給 gate/council。  

主要路徑：

- `tonesoul/semantic_control.py`
- `tonesoul/tension_engine.py`

## L3 Reasoning & Deliberation

責任：

1. 多視角審議與投票。  
2. 生成 verdict 候選與結構化理由。  

主要路徑：

- `tonesoul/council/pre_output_council.py`
- `tonesoul/council/perspective_factory.py`
- `tonesoul/council/types.py`
- `tonesoul/council/verdict.py`

## L4 Audit & Explainability

責任：

1. 把決策變成可追溯、可審計的結構化輸出。  
2. 明示不確定性、分歧與責任欄位。  

主要路徑：

- `tonesoul/council/transcript.py`
- `tonesoul/council/summary_generator.py`
- `tonesoul/contract_observer.py`
- `tests/test_uncertainty.py`
- `tests/test_verdict.py`

## L5 Governance Kernel

責任：

1. 以 P-level + POAV + DCS 執行門控。  
2. 輸出 `PASS/REWRITE/BLOCK` 或升級決策。  

主要路徑：

- `tonesoul/yss_gates.py`
- `tonesoul/yss_pipeline.py`
- `tonesoul/poav.py`
- `tonesoul/dcs.py`
- `tonesoul/escalation.py`

## L6 Narrative Continuity

責任：

1. 保持可回放的決策與記錄格式。  
2. 將長期治理證據維持在可稽核狀態。  

主要路徑：

- `law/step_ledger_schema.json`
- `docs/STEP_LEDGER_SPEC.md`
- `docs/engineering/EXAMPLES/step_ledger_example.md`
- `tonesoul/memory/*`（記憶聚合與回溯）

## 3. 執行流程（簡化）

1. L1 接收輸入  
2. L2 產生向量訊號  
3. L3 形成多視角 verdict  
4. L5 進行 gate 決策  
5. L4/L6 輸出可追溯資料  
6. 若需重寫，回到 L3 或 L1 前置步驟

## 4. 與舊版差異

1. 移除過時 `body/*.py` 對位。  
2. 全部改為本 repo 實際可定位路徑。  
3. 明確區分 L4（可解釋輸出）與 L6（記錄連續性）。  

## 5. 驗收條件

1. 文件內每條程式路徑 `Test-Path == True`。  
2. 六層至少各有一個實際模組對位。  
3. 不再混用 `ΔS` 漂移語義（漂移統一用 `ΔΣ`）。
