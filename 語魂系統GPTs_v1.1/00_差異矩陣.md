# 語魂系統雙版本差異矩陣（Phase 1）

日期：2026-02-11  
範圍：`語魂系統GPTs/` vs `語魂系統GPTs_Codex版/`

## 1. 總覽判定

- 原版優勢：哲學敘事完整、研究索引與 `SYSTEM_PROMPT` 已成形。
- Codex 版優勢：術語單一化、Runtime/Human 分層、測試與審計格式可直接落地。
- 主要風險：兩版若直接混用，會出現術語衝突、門檻歧義、路徑不可追溯。

## 3. 關鍵衝突（需在 v1.1 收斂）

## 3.1 術語衝突：`ΔS` 與 `ΔΣ`

- 現象：
  - 原版同時存在「`ΔS` 是 direction」與「`ΔS` 是 drift」語境。
  - 原版部分文件又宣告 `ΔS` 可能落在 `[-1,1]` 或 `[0,1]`。
- 影響：
  - 相同輸入在不同章節可能觸發不同 Gate 判定。
- 收斂規則：
  - `ΔS` 固定為 direction（`[-1,1]`）。
  - `ΔΣ` 固定為 divergence（`[0,1]`）。
  - 舊文「ΔS 漂移」統一改寫為「ΔΣ 漂移」。

## 3.2 POAV 雙定義衝突

- 現象：
  - 原版同時允許兩套 POAV 解讀（`Precision/Observation/Avoidance/Verification` 與 `Parsimony/Orthogonality/Audibility/Verifiability`）。
  - 原版另有 `Precision = 1 - (字數/最大字數)` 的高風險公式。
- 影響：
  - 指標含義不唯一，審計分數不可比較。
  - 可能鼓勵「短但不準」輸出。
- 收斂規則：
  - v1.1 採單一定義（以 Codex 版 01 為準）。
  - 移除字數反推 precision 的公式。

## 3.3 追溯風險：失效路徑引用

- 檢查結果（正則掃描）：
  - 原版：`Exists=True 36`、`Exists=False 10`
  - Codex 版：未檢出失效檔案路徑引用
- 主要失效來源：
  - `語魂系統GPTs/04_六層架構與系統設計.txt` 內 `body/*.py` 路徑。
- 影響：
  - 文件宣稱無法映射到實際 repo，審計不可回放。
- 收斂規則：
  - v1.1 全部改為當前 repo 實際可定位路徑。

## 4. 合併策略（v1.1）

1. 規格骨架：採 `語魂系統GPTs_Codex版/01~06`
2. 敘事主體：移植原版 `01/02/03/08/09/15`，但術語與門檻全部對齊單一規格
3. Prompt 與研究索引：保留原版 `SYSTEM_PROMPT`、`11_相關研究論文索引`，先做一致化再納入
4. 審計最小鏈：每個核心命題都需有「行為 -> 測試 -> 證據」三欄對位
