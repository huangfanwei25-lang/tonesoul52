# 語魂簡報一頁圖（文字版，v1.1）

日期：2026-02-11  
狀態：`presentation-ready`

## 一句話定位

語魂不是在模仿人格，而是在把「誠實、邊界、責任」做成可驗證的系統流程。

## 一頁流程圖（文字版）

使用者輸入  
↓  
語義感知（`ΔT / ΔS / ΔΣ / ΔR`）  
↓  
多視角審議（`Muse / Logos / Aegis`）  
↓  
Gate 決策（`P0/P1/P2 + POAV`）  
├─ `PASS` -> 輸出回覆  
├─ `REWRITE` -> 重寫（最多 3 次）  
└─ `BLOCK` -> 拒絕 + 安全替代路徑  
↓  
StepLedger（append-only 審計記錄）

## 核心判定規則（簡報表）

`P0`（不傷害）：
- 命中危害紅線，直接 `BLOCK`。

`P1`（事實與誠實）：
- 不得亂編、不得裝懂、不得偽裝能力。

`P2`（意圖對齊）：
- 在不違反 `P0/P1` 前提下，提供可執行幫助。

`POAV` 門檻：
- `POAV >= 0.70` -> `PASS`
- `0.30~0.69` -> `REWRITE`
- `< 0.30` -> `BLOCK`
- 高風險任務 -> 需 `>= 0.90`

## 三個現場重點

1. 可追溯：每次決策都有紀錄，不是黑箱回答。  
2. 可收斂：回答不好會重寫，不是一次失誤就直接放行。  
3. 可問責：不是看語氣好不好，而是看規則是否被滿足。

## 七悖論對應（超精簡版）

- `PX-001` 自傷請求：`BLOCK`（`P0`）
- `PX-002` 危險知識：`BLOCK`（`P0`）
- `PX-003` 秘密追蹤：`BLOCK/REWRITE`（`P0/P1`）
- `PX-004` 商業侵權：`REWRITE/BLOCK`（`P1/P2`）
- `PX-005` 偏見背書：`BLOCK`（`P0/P1`）
- `PX-006` 火場求生：`PASS`（`P0` 緊急子條款）
- `PX-007` 情緒依附：`REWRITE`（邊界 + 支持）

## 收束句（結尾可用）

語魂不是追求「最會說話」，
而是追求「說出口的每一句，都能被追問、被驗證、被負責」。
