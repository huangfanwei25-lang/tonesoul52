# 語魂術語與門檻單一規格（v1.1）

日期：2026-02-11  
狀態：`authoritative`（本文件為術語與門檻唯一真相來源）  
適用範圍：`語魂系統GPTs_v1.1/*`

## 0. 規格優先級

1. 術語、門檻、優先級衝突時，以本文件為準。  
2. 舊文件若與本文件不一致，視為 legacy，需在遷移說明中標註。  
3. 不允許同一概念存在兩套定義。

## 1. 核心向量（唯一版本）

1. `DeltaT (ΔT, Tension)`
- 定義：情緒/互動張力強度
- 範圍：`[0, 1]`

2. `DeltaS (ΔS, Direction)`
- 定義：語義方向/立場極性（推進或收斂）
- 範圍：`[-1, 1]`
- 注意：`ΔS` 不是漂移分數。

3. `DeltaSigma (ΔΣ, Semantic Divergence)`
- 定義：語義偏離/散度（相對語境錨點的偏離）
- 範圍：`[0, 1]`
- 注意：所有「語義漂移」都使用 `ΔΣ` 表達。

4. `DeltaR (ΔR, Risk Density)`
- 定義：風險密度/危害機率綜合分數
- 範圍：`[0, 1]`

## 2. Legacy 映射（過渡規則）

1. 舊文寫法「`ΔS 漂移`」一律映射為「`ΔΣ 漂移`」。  
2. `tonesoul/semantic_control.py` 內部仍以 `delta_s` 表示 divergence，並提供 `delta_sigma` alias；文件層統一稱 `ΔΣ`。  
3. 任一文件不可同時宣告 `ΔS` 具有 `[-1,1]` 與 `[0,1]` 兩種範圍。

## 3. P-level 優先級（唯一版本）

- `P0`: Non-Harm（不傷害硬紅線）  
- `P1`: Factual Accuracy + Honesty（事實準確與誠實）  
- `P2`: Intent Alignment（意圖對齊與可用性）  
- `P3`: Efficiency（效率與結構清晰）  
- `P4`: Style / Preference（風格與偏好）

規則：高優先級永遠覆蓋低優先級。

## 4. Gate 輸出語義

- `PASS`: 可放行  
- `REWRITE`: 必須重寫後再審  
- `BLOCK`: 不可放行，需提供安全替代路徑

## 5. POAV（唯一版本）

`POAV = (P + O + A + V) / 4`

- `P = Parsimony`（簡潔且不遺漏必要資訊）
- `O = Orthogonality`（結構去重、語義不重疊）
- `A = Audibility`（推理可解釋）
- `V = Verifiability`（可查證、可審計、可追溯）

### 5.1 基準門檻（產品規格）

- `POAV >= 0.70` -> `PASS`
- `0.30 <= POAV < 0.70` -> `REWRITE`
- `POAV < 0.30` -> `BLOCK`
- 高風險任務：`POAV >= 0.90` 才可 `PASS`

### 5.2 與當前程式的相容說明

- `tonesoul/yss_gates.py` 的 `poav_gate()` 在 `enforce=false` 時採 `record_only`，在 `enforce=true` 時才會直接 `block`。  
- `REWRITE` 層通常由上層控制流程執行（非 `poav_gate()` 單點完成）。

## 6. 向量觸發門檻（建議基線）

- `ΔR > 0.95` -> `BLOCK`（P0）  
- `0.70 < ΔR <= 0.95` -> 高敏審查  
- `ΔΣ > 0.80` -> 要求語境重確認/重寫  
- `ΔT > 0.60` -> 啟用降溫語氣策略

## 7. 緊急例外（P0 子條款）

當即時生命威脅成立且回覆目的是救命：

1. 可暫時放寬部分 `P1/P2` 約束。  
2. 不可放寬 `P0`（不得轉化為傷害指引）。  
3. 必須寫入可追溯審計紀錄（至少含決策理由與時間戳）。

## 8. 禁用寫法（強制）

1. 禁止同時宣告 `ΔS` 兩種數值範圍。  
2. 禁止把「語義漂移」命名為 `ΔS`。  
3. 禁止同時保留兩套 POAV 詞義。  
4. 禁止使用「字數越短越精確」作為 Precision/Parsimony 的唯一判定依據。
